# AI能力边界

设计AI角色的基础认知。

## 设计风险

- **过度承诺**：设计了AI做不到的能力
- **过度保守**：没有充分发挥AI的优势
- **定位错误**：让AI做不擅长的事，让人做重复劳动

## 能力概览

| 能力 | AI强项 | AI局限 |
|------|--------|--------|
| 语言理解生成 | 流畅、多语言、上下文 | 专业晦涩内容可能有偏差 |
| 知识检索整合 | 极快、多源整合 | 受限于训练数据 |
| 模式识别 | 准确、快速、大规模 | 训练数据外识别有限 |
| 逻辑推理 | 复杂链、一致性 | 缺乏常识，可能产生貌似合理的错误 |
| 即时响应 | 毫秒级、高并发 | 复杂问题需更长时间 |
| 数据分析 | 海量、发现隐藏模式 | 可能发现虚假相关性 |

## 人机分工原则

**AI负责**：标准化、重复性、大数据、快速响应、模式识别
**人类负责**：复杂判断、创造性、情感连接、异常处理、价值判断

## 设计检查清单

**能力边界**
- [ ] 所有设计的能力AI都能实现？
- [ ] 没有过度承诺？
- [ ] 明确了局限性？

**人机分工**
- [ ] AI负责擅长的任务？
- [ ] 转接机制清晰？

**风险控制**
- [ ] 识别了AI可能出错的场景？
- [ ] 有兜底方案？

## 常见错误

| 错误 | 正确做法 |
|------|----------|
| 让AI做复杂情感咨询 | AI整理信息，人类深度咨询 |
| 认为AI只能做简单FAQ | 发挥AI分析、记忆、学习能力 |
| 让AI自主做所有决策 | 关键节点设置人类审核 |
| 模糊AI和人类职责 | 明确转接条件 |
