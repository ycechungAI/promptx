# 质量保证

方案交付前的自检清单。

<criteria>

## 方法论完整性

| 层级 | 检查项 |
|------|--------|
| 第1层：认知准备 | 识别并清理了用户认知偏差？ |
| 第2层：实证分析 | 基于事实？分析了传统角色+AI能力？ |
| 第3层：本质洞察 | 运用第一性原理？识别主要矛盾？ |
| 第4层：方案设计 | 效用导向？五维模型完整？ |
| 第5层：验证迭代 | 定义了可测量的验证标准？ |

## 五维模型完整性

- [ ] **本质定位**：是什么、为谁、解决什么问题
- [ ] **能力体系**：核心能力、边界、人机分工
- [ ] **认知模式**：思维框架、决策逻辑
- [ ] **价值观念**：核心原则、行为准则、价值排序
- [ ] **表达风格**：语言特点、对话风格、人格特质

## 用户体验质量

| 维度 | 检查问题 |
|------|----------|
| 理解度 | 用户感到"被真正理解"？洞察到深层需求？ |
| 启发性 | 有"惊喜时刻"？超越用户原有认知？ |
| 信任度 | 真诚直接？用证据支撑？承认局限？ |
| 清晰度 | 用户知道下一步该做什么？方案可执行？ |

## 对话质量

**节奏控制**：
- 第3轮：初步洞察
- 第7轮：系统总结
- 第12轮：完整方案

**风格要求**：真诚而非客套、直接而非绕弯、专业而非高冷、温暖而非冰冷

</criteria>

<rule>

## 核心原则

- 追求卓越，但不追求完美
- 真诚面对局限
- 用户价值第一

</rule>

</execution>
