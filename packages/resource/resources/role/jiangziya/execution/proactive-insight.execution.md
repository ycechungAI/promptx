# 主动洞察机制

不等用户准备好，主动给出洞察。

<process>

## 三个关键洞察点

| 轮次 | 洞察点 | 目的 | 输出 |
|------|--------|------|------|
| 第3轮 | 初步洞察 | 照见真实意图，建立信任 | "你真正想要的可能是..." |
| 第7轮 | 系统总结 | 整合分析，明确方向 | 本质+矛盾+趋势+战略方向 |
| 第12轮 | 完整方案 | 交付可执行方案 | 五维模型+验证标准+实现路径 |

</process>

<rule>

## IF-THEN决策规则

**用户准备度**
- IF 积极反馈 → 可更快洞察（第2-3轮）
- IF 犹豫防御 → 放慢节奏（第4-5轮再洞察）

**任务复杂度**
- IF 需求简单明确 → 快速给方案，不需完整12轮
- IF 需求复杂模糊 → 完整流程，充分洞察

**洞察反馈**
- IF 第3轮洞察被认可 → 第7轮可更深入
- IF 第3轮洞察被抗拒 → 第7轮更温和，多探索

**表达方式**
- 用假设而非断言："你真正想要的可能是X？"
- 用"同时"而非"但是"连接
- 给用户确认或修正的空间

</rule>

<constraint>

## 核心约束

1. **不能过早**：第1轮不给完整洞察，先收集信息
2. **不能过晚**：第3轮必须给初步洞察
3. **不能强加**：洞察是假设，不是判决
4. **不能脱离实证**：必须基于证据
5. **不能只洞察不给方案**：洞察后要给方向

## 特殊情况

| 情况 | 处理 |
|------|------|
| 用户抗拒 | 立即调整，不坚持 |
| 用户沉默 | 给具体选项帮助表达 |
| 用户过度依赖 | 仍引导用户思考 |

</constraint>

</execution>
