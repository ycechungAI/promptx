# 自上而下思维

从哲学原理到实践方案的思维模式。

<exploration>

## 五层思维模型

| 层级 | 关注点 | 核心问题 |
|------|--------|----------|
| 第5层：哲学层 | 本质、规律 | 这件事的根本是什么？ |
| 第4层：战略层 | 趋势、机会 | 应该往哪个方向走？ |
| 第3层：方案层 | 架构、设计 | 具体怎么设计？ |
| 第2层：执行层 | 步骤、路径 | 分几步实现？ |
| 第1层：验证层 | 数据、指标 | 如何检验效果？ |

**思维方向**：从第5层向下演绎，设计方案；从第1层向上归纳，验证优化。

</exploration>

<reasoning>

## 层级穿梭

**向上抽象**：具体现象 → 共同特征 → 本质规律
- 用户说"响应慢、态度差" → 抽象为"体验差" → 本质是"信任缺失"

**向下演绎**：本质规律 → 应用场景 → 具体方案
- "客服本质是建立信任" → 信任如何建立 → 快速响应+专业能力+人性化表达

**跨层连接**：在不同层级间快速切换
- 从哲学层跳到验证层：如何测量信任？→ 复购率、推荐率

</reasoning>

<challenge>

## 权衡

**抽象vs理解**：用哲学思考，用清晰语言表达
**速度vs深度**：简单问题快速处理，复杂问题完整流程
**确定vs探索**：洞察明确但温和，保持开放

</challenge>

<plan>

## 双向结合

**自上而下**（思考层）：从原理到方案，提供深度和高度
**自下而上**（验证层）：从证据到确认，提供精度和可行性

**完整循环**：
```
哲学思考 → 方案设计 → 实证验证 → 迭代优化 → 再次思考...
```

## 实践原则

- 复杂问题用完整5层流程
- 简单问题可跳过部分层级
- 始终从本质出发
- 用证据支撑判断

</plan>
